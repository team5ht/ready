# Syllable Trainer — iteration 2

Вторая UX-итерация после разрезания файлов и первой правки детского экрана.

Цель: основные элементы управления больше не требуют чтения. Ребёнок видит крупные символы, а текстовые подписи сохранены для доступности и взрослого понимания через `aria-label` и скрытый `.sr-only`-текст.

## Что изменено

- `Начать` заменено на большую кнопку `▶`.
- `Слушай` заменено на `🔊`.
- `Дальше` заменено на `➜`.
- `Ещё раз` заменено на `↻`.
- `Подсказка / ?` заменено на `💡`.
- Добавлен CSS-класс `.sr-only` для скрытых текстовых подписей.
- Логика кнопки запуска обновлена так, чтобы временное состояние загрузки тоже было без видимого текста: `⏳` + скрытая подпись `Загрузка`.

## Структура

```text
index.html
css/styles.css
js/config.js
js/audio.js
js/game.js
js/drag.js
js/main.js
audio/
```

Файлы аудио должны лежать в `audio/` рядом с `index.html`, например:

```text
audio/make_syllable.mp3
audio/need.mp3
audio/success.mp3
audio/ma.mp3
...
```

Порядок подключения JS-файлов в `index.html` важен:

```html
<script src="js/config.js"></script>
<script src="js/audio.js"></script>
<script src="js/game.js"></script>
<script src="js/drag.js"></script>
<script src="js/main.js"></script>
```

## Проверка

1. Положить аудиофайлы в папку `audio/`.
2. Открыть `index.html` через локальный сервер или GitHub Pages.
3. Проверить старт, повтор задания, подсказку, правильный/неправильный drag, экран завершения и кнопки `↻` / `➜`.

Аудиофайлы в архив не включены.
